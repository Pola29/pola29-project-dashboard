# Iteration（スプリント）の扱い

**Project v2 の Iteration フィールド**を、スプリント（週次の箱）の**正**とする。

## 推奨の粒度

- **1 週間**＝1 Iteration、またはチーム合意の **2 週間**＝1 Iteration。  
- 先に**どちらか一つ**に揃え、週次で見直す。

## 命名の例

| スタイル | 例 |
|----------|-----|
| ISO 週 | `2026-W17` |
| 日付範囲 | `2026-04-21..04-25`（短い期間用） |
| 連番 | `Sprint-2026-12`（四半期や通し番号を別途管理する場合） |

**ルールを 1 本に固定**し、Project の**ドロップダウン**に出てくる表記を全員同じにする。

## 運用

- **新規 Issue** を Project に乗せたら、**未設定の Iteration** をゼロに近づける（週次またはデイリーで直す）。  
- **前の週**の Iteration だけが延々残らないよう、**Done** への移動とセットで意識する。  
- 週の途中で**翌週に回す**場合は、**Iteration を付け替える**（ラベルと二重管理しない）。

## Web での作成

Org → **Projects** → 対象 Project → **Settings**（または **Iteration** フィールドの管理）で、**未来の週**を先に数件作っておくと、週始めの負担が減る。

## `gh` について

Iteration の一括操作は、GitHub API / `gh` の対応状況に依存する。最低限、**Web で週次を作る**＋**Issue ごとに Iteration 割り当て**で回し、`gh` は Project / Issue の紐づけに使う、で十分。詳細は `../scripts/gh-project-snippets.sh` のコメントを参照。
